# PrismaX Narrative

The canonical narrative content for prismax.ai (V1 rebuild). These files capture the positioning, messaging, and voice that drive every page of the site.

Finalized 2026-05-12 by Chyna. Pull from these files when writing or reviewing site copy.

## Files

| File | Purpose |
|---|---|
| [foundation.md](foundation.md) | Positioning statement, category definition, the core problem, why now, how PrismaX fits |
| [strategic-narrative.md](strategic-narrative.md) | Strategic narrative, how PrismaX works, the role of the community |
| [descriptions.md](descriptions.md) | One-paragraph, 50-word, full, and shortened boilerplates |
| [elevator-pitch.md](elevator-pitch.md) | Elevator pitch + audience-specific CTAs |
| [key-messages.md](key-messages.md) | Core message, what matters, proof points — by audience |
| [mission-vision.md](mission-vision.md) | Mission and vision statements |
| [pillars.md](pillars.md) | The three product pillars — Robots / Data / Intelligence |
| [voice.md](voice.md) | Voice principles, how PrismaX sounds, internal narrative anchors |

## Use

1. Match every page or section to the relevant audience tier — see [key-messages.md](key-messages.md).
2. Pull verbatim from [descriptions.md](descriptions.md) for boilerplates. Do not paraphrase.
3. Apply the voice principles in [voice.md](voice.md) — especially the **Good** vs. **Avoid** examples.
4. Anchor each section to a pillar in [pillars.md](pillars.md) or one of the narrative anchors in [voice.md](voice.md).
5. Numbers: V1 launches under a no-numbers posture (see [HOMEPAGE.md](../HOMEPAGE.md)) — no proof bar, no leading with stats. Specific figures that do appear in boilerplates (e.g. seed-round amount in [descriptions.md](descriptions.md)) are maintained in the canonical PrismaX marketing repo and synced into this site manually when they change.
